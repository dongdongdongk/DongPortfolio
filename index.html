공통 기능 및 유틸리티
  • Tailwind CSS를 사용하여 디자인 작업을 진행했습니다. 클래스 기반의 스타일링은 일관된 디자인을 적용하는 데 용이했습니다. 클래스명을 기억해야 하는 학습 곡선이 있었지만, 익숙해지면서 생산성이 크게 향상되었습니다.
  • ErrorHandler 클래스를 정의하여 다양한 에러를 효율적으로 처리할 수 있었습니다. 통일된 형태의 에러 객체를 생성하여 에러 핸들링을 보다 일관되게 할 수 있었습니다.
  • 전역 에러 핸들러 미들웨어를 구현하여 다양한 종류의 에러에 대한 적절한 응답을 생성하게 했습니다. 또한, catchAsyncErrors 미들웨어를 작성하여 비동기 함수에서 발생하는 에러를 효과적으로 처리할 수 있었습니다.
  • Multer를 활용하여 파일 업로드를 간편하게 처리할 수 있었습니다.
  • React-Toastify를 도입하여 액션 완료나 에러 발생 시 사용자에게 적절한 피드백을 주어 사용자 경험을 향상시킬 수 있었습니다.
  • React.lazy()와 Suspense를 사용하여 코드를 런타임에서 분할하여 초기 로딩 속도를 개선하였습니다. 웹 브라우저의 Performance 탭을 이용하여 첫 페이지 로드 완료 시점(Load Event)까지의 시간을 초기 로딩 시간으로 측정한 결과, 적용 전에는 평균 2.5초였던 초기 로딩 시간이 적용 후에는 1.8초로 측정되어 0.7초, 기존 대비 28%의 로딩 시간 단축 효과가 있었습니다.
  • ProtectedRoute 컴포넌트를 생성하여 해당 라우트에 접근하기 위한 권한 확인 로직을 구현했습니다. 이를 통해 인증된 사용자만이 허용된 페이지에 접근할 수 있도록 보안을 강화할 수 있었습니다.
  • 로더 컴포넌트를 만들고 Lottie 등의 애니메이션을 적용하여 데이터 로딩이나 처리과정에서 사용자에게 직관적인 시각적 피드백을 제공하여 사용성을 향상시켰습니다.
링크 인증을 통한 회원가입 프로세스 구현
  • 사용자가 회원가입 양식을 제출하면, 서버는 사용자의 이메일 주소로 확인 링크를 포함한 이메일을 보냅니다.
  • 이 링크에는 사용자를 고유하게 식별할 수 있는 JWT 토큰이 포함되어 있습니다.
  • 사용자가 이메일을 받고 확인 링크를 클릭하면, 서버는 해당 JWT 토큰을 검증합니다. JWT 토큰이 유효하면, 해당 사용자를 확인하고 회원가입 프로세스를 완료합니다.
  • Nodemailer를 사용하여 이메일을 전송하고, JWT를 생성하고 검증하는 jsonwebtoken 라이브러리를 활용하였습니다.
  • 비밀번호는 BCrypt를 활용한 해싱을 통해 암호화되어 안전하게 저장되었습니다.
찜 & 장바구니 추가기능
  • 사용자가 상품을 찜하기 또는 장바구니에 추가할 때, localStorage를 이용하여 상품 정보가 저장됩니다.
  • 사이드 바에는 localStorage데이터를 이용하여 찜 목록과 장바구니 내역을 표시합니다.
  • 팝업을 띄우기 위해 조건부 렌더링을 사용하였습니다.
  • 팝업을 띄우는데 포탈(Portal)을 이용하는 대신 조건부 렌더링(Conditional Rendering)을 통해 팝업을 제어하는 방식을 사용하였습니다. 이 선택은 앱의 규모와 복잡성을 고려하여 결정하였습니다.
  • 카트에 담긴 총 상품 가격을 계산하기 위해 할인율과 수량에 따른 계산을 reduce를 사용하여 수행했습니다.
사용자 주소 및 결제 기능
  • 사용자는 개인 프로필 페이지에서 여러 주소를 등록할 수 있습니다. 주소는 집이나 회사 등 다양한 목적으로 등록 가능합니다.
  • 주소 등록 시 country-state-city 라이브러리를 활용하여 국가, 주(State), 도시(City) 정보를 선택할 수 있도록 했습니다.
  • 결제 시 주소를 반복해서 입력하는 번거로움을 해소하기 위해, 전역 상태로 관리되는 addresses 배열을 Redux를 통해 구현했습니다. 사용자는 등록한 주소 목록에서 편리하게 선택할 수 있습니다.
  • localstorage의 lastOrder에 장바구니 정보를 저장해두고 결제 과정에서 이를 불러와 사용합니다. 이를 통해 결제 과정 중에도 장바구니 정보를 유지할 수 있습니다.
  • 결제 완료 시 장바구니 데이터를 삭제하고 결제 정보를 서버에 저장하여 배송 추적 등 후속 작업에 활용했습니다.
  • 쿠폰 코드 입력 시 유효성 검사를 통해 유효하지 않은 쿠폰을 방지하고, 할인된 가격을 계산할 때 reduce를 사용하여 코드를 효율적으로 작성했습니다.
이벤트 상품 및 실시간 카운터 기능
  • 판매자는 혜택이 있는 이벤트 상품을 등록할 수 있습니다. useRef 훅을 사용하여 end-date 요소에 접근하여 min 속성의 값을 설정하여 최소 종료일 로직을 구현했습니다.
  • 최소 종료일을 지정함으로써 유효하지 않은 날짜 입력을 방지하고, 이벤트 상품의 종료일이 적절한 기간 내에 있도록 보장할 수 있게 되었습니다.
  • CountDown 컴포넌트를 제작하여 실시간으로 남은 이벤트 시간이 출력되도록 하였습니다. 카운터 컴포넌트는 날짜 데이터를 의존성 배열에 설정하여 변경될 때마다 useEffect 함수가 호출되어 갱신되도록 설정하였습니다.
판매자 & 구매자 실시간 1대1 채팅 기능
  • 사용자는 판매자에게 1대1 채팅으로 상품에 대한 문의를 할 수 있습니다. 이를 위해 웹 소켓을 활용하여 채팅 기능을 구현하였습니다.
  • 기존 서버 외에 독립적인 소켓 서버를 새로 만들었습니다.결과적으로 채팅을 위한 전용 리소스를 할당하고 성능을 최적화할 수 있었습니다. 또한 HTTP 트래픽과 소켓 트래픽을 분리함으로써 서버의 안정성과 확장성을 높일 수 있었습니다.
  • 메시지의 생성 시간을 표시하기 위해 `timeago.js`를 사용하였습니다. 상대적인 시간 정보를 표시하여 사용자에게 보다 직관적인 정보를 제공할 수 있었습니다.
  • 모든 채팅 내용은 서버로 전송되어 저장되며 이를 통해 사용자는 이전 대화 기록을 다시 확인할 수 있습니다.
  • 채팅방에는 마지막 메시지를 출력해줍니다. 또한, 사용자 자가 온라인 상태인지를 체크하여 상태를 표시합니다.

🌏 [트러블 슈팅]

  • 프론트의 index.js 에 React.StrictMode 가 있으면 사용자가 두번 등록되는 문제 발생
  • 프론트의 index.js 파일에 React.StrictMode가 포함되어 있으면 개발 모드에서 사용자 등록 과정에서 중복 등록되는 문제가 발생하였습니다.(React.StrictMode가 자식 컴포넌트를 두 번 렌더링하여 잠재적인 부작용을 확인함 이로 인해 상태 변경이나 사이드 이펙트가 두 번씩 발생하는 문제 )
  • 해결 : React 공식 문서와 온라인 커뮤니티를 탐색했습니다. 검색어 ("React duplicate render", "React.StrictMode double rendering" 등)
  • React.StrictMode가 개발 모드에서 자식 컴포넌트를 두 번 렌더링하여 잠재적인 부작용을 확인한다는 내용을 발견했습니다.React.StrictMode를 주석 처리하거나 삭제하는 방식으로 문제를 해결했습니다.
  • 상품 이름에 공백이 포함되어 있어서 변수 처리 시 오류가 발생함.
  • 해결: 변수에 저장된 제품 이름에서 공백을 모두 "-"로 대체하도록 정규 표현식을 사용하여 처리했습니다. 정규 표현식 / /g를 활용하여 모든 공백을 찾아서 "-"로 대체하였습니다.
  • 사용자 전화번호가 누락되서 저장되는 문제 발생
  • 해결 : 모델에서 사용자의 전화번호 필드를 Number 타입으로 정의했던 것이 문제였습니다.( 숫자의 제일 첫 0 을 제거해버림 ) JavaScript에서는 일부 번호가 정수로 해석될 때 정밀도 손실이 발생할 수 있기 때문에 이러한 문제가 발생했습니다. 따라서 전화번호 필드를 String 타입으로 변경하여 해결하였습니다.
  • 웹 애플리케이션에서 외부 도메인으로부터 데이터를 요청할 때 CORS 오류가 발생
  • 외부 API로부터 데이터를 가져오는 과정에서 CORS(Cross-Origin Resource Sharing) 오류가 발생하였습니다. 이는 웹 브라우저의 동일 출처 정책(Same-Origin Policy)으로 인해 발생한 문제로, 보안상의 이유로 다른 도메인에서 오는 요청을 제한하는 정책이였습니다.
  • 해결: 이 문제를 해결하기 위해 먼저 CORS 오류의 원인과 동작 방식에 대해 MDN 웹 문서와 관련 자료를 참고하여 이해하고자 했습니다. 그 후 여러 해결 방법 중 cors 라이브러리를 사용하기로 결정했습니다

MyPlaces 프로젝트 상세 내용

📌 [개발 환경]

  • O.S/ Windows 10
  • 언어/ JavaScript
  • DBMS/ MongoDB

    • IDE : VSCode

🏠 [프로젝트 주요 기능]

  • 사용자가 이미지와 함께 공유하고자 하는 장소의 주소를 제출하면, 구글맵을 활용하여 해당 장소를 시각적으로 표시하는 기능을 구현하였습니다.
  • 로그인 및 자동 로그아웃 기능은 JWT 리프레쉬 토큰과 엑티브 토큰을 활용하여 구현하였습니다.
  • 다양한 리액트 훅(useCallback, useReducer, 커스텀 훅)을 활용하여 기능들을 효율적으로 구현하였습니다.

⛽ [프로젝트 상세 설명]

공통 기능 및 유틸리티
  • 반복적으로 사용되는 API 호출 로직에서 많은 중복이 생겼습니다. API 호출 로직을 커스텀 훅으로 분리하여 코드를 리팩토링을 진행 중복 문제를 해결하였습니다.
  • 포털을 도입 팝업창을 별도의 DOM 요소로 렌더링하여 재사용성과 불필요한 랜더링을 줄여 성능을 향상시켰습니다.
  • 쿠키를 사용하여 JWT를 관리하는 방식이 보안 측면에서 일부 취약점을 가지고 있었습니다. 이를 해결하기 위해 헤더에 JWT를 첨부하는 방식을 도입하여 보안을 강화하였습니다.
JWT 를 활용한 로그인 & 자동 로그아웃 구현
  • 직접 커스텀 input 컴포넌트를 만들어 여러 컴포넌트에서 재 사용하도록 하였습니다.
  • useReducer를 활용하여 여러 상태를 동시에 처리하며 코드의 복잡성을 줄였습니다.
  • 직접 작성한 validatiors.js를 작성하여 form의 유효성 검사를 수행하여 외부 라이브러리에 의존하지 않고도 유효성 검사를 실시할 수 있었습니다.
  • useCallback 훅을 활용하여 메모이제이션 기능을 이용하여 불필요한 메모리 사용을 줄이고, 렌더링이 불필요하게 발생하는 것을 방지하여 성능을 향상시켰습니다.
  • 로그인 기능을 위해 jsonwebtoken을 사용하여 JWT 토큰을 생성하였고, 보안을 강화하기 위해 회원 가입 시 bcryptjs 라이브러리를 이용하여 비밀번호를 해싱하였습니다.
  • 로그아웃 기능은 logoutTimer 변수를 전역 변수로 선언하여 토큰 만료 시간과 현재 시간의 차이를 계산하고, setTimeout을 사용하여 자동으로 로그아웃되도록 설정하였습니다.
장소 관리 & 커스텀 훅을 이용한 리팩토링
  • 사용자가 장소를 추가할 수 있습니다. 추가된 장소는 구글 맵 API를 활용하여 입력한 주소를 기반으로 위치를 지도에 표시합니다.
  • 사용자는 추가한 장소를 확인할 수 있습니다. 이를 통해 장소의 정보를 확인하고 수정 또는 삭제할 수 있습니다.
  • 유효성 검사와 초기값 설정이 장소 추가 및 수정 폼에서 중복되어 코드의 가독성과 유지 보수성이 떨어졌습니다.
  • 문제를 해결을 위해 리팩토링을 진행(커스텀 훅 form-hook.js를 작성) 중복 코드를 피하고 코드의 재사용성을 높였습니다.

🌏 [트러블 슈팅]

  • 사용자 프로필 이미지가 올바른 URL을 사용하더라도 표시되지 않는 문제가 발생함.
  • 해결: 문제는 express 애플리케이션에서 정적 파일을 제공하지 않아 발생한 것으로 파악되었습니다. 이를 해결하기 위해 express 애플리케이션에 express.static 미들웨어를 추가하여 정적 파일을 서빙하도록 설정했습니다. 이를 통해 사용자 프로필 이미지 파일이 올바르게 제공되고 표시될 수 있게 되었습니다.
  • 토큰 검증 미들웨어를 사용할 때 OPTIONS 오류 발생
  • 해결: 클라이언트에서 발생한 OPTIONS 요청이 올바르게 처리되지 않아 발생했습니다. 이를 해결하기 위해 토큰 검증 미들웨어에서 OPTIONS 요청을 처리할 수 있도록 조건문을 추가하여 문제를 해결하였습니다. ( if(req.method === 'OPTIONS') )

회원페이지 & 찜기능
  • 회원이 방문한 가게 정보를 데이터베이스에 저장하도록 설계 했습니다. 방문한 가게의 정보를 유지하고 관리하여 회원에게 개인화된 서비스를 제공합니다.
  • 로그인한 회원이 찜하기 버튼을 클릭하면 WishList 테이블에 해당 정보를 추가하도록 구현했습니다. 이를 통해 회원이 찜한 가게 목록을 관리하고 조회할 수 있습니다.
  • 크롤링한 페이지의 URL을 데이터베이스에 저장하여 즐겨찾기 목록에서 클릭 시 해당 URL을 사용하여 페이지를 동적으로 생성할 수 있도록 구현했습니다.

🌏 [트러블 슈팅]

  • Geocoder가 내 위치를 파악하지 못했을 때 오류 발생
  • 해결: Geocoder가 위치를 파악하지 못했을 때에 대비하여, 실패 시 사용할 기본 위치를 지정했습니다. Geocoder 요청을 보낸 후에, 응답이 실패하면 기본 위치로 서울의 좌표를 사용하여 처리하도록 설정했습니다. 이를 통해 Geocoder가 사용자의 위치를 파악하지 못했을 때에도 정상적으로 처리할 수 있도록 문제를 해결했습니다.
  • 크롤링하는 페이지나 사이트의 형식이 다양하여, 데이터를 찾지 못했을 때 오류가 발생
  • 해결: 이러한 문제를 해결하기 위해 크롤링 과정에서 알맞은 형식의 데이터를 찾는 비즈니스 로직을 구현했습니다. 이를 통해 각 페이지나 사이트의 형식에 따라 데이터를 적절히 파싱하여 정확한 정보를 추출할 수 있도록 했습니다.
  • 크롤링 사용 시 페이지 로딩 속도 저하 문제 발생
  • 해결: 코드의 통일성을 위해 동적 크롤링 (Selenium) 하나로 크롤링을 진행하려 하였지만 페이지 로딩속도 저하가 발생하여 Jsoup 과 Selenium 을 혼합하여 사용하여 페이지 로딩 속도를 개선시켰습니다.
  • 크롤링하는 페이지의 테이블 형식이 다양하여, 오류가 발생
  • 해결: 각 페이지의 테이블 형식을 파악하여 알맞은 테이블 엘리먼츠를 선택하고 파싱하는 비즈니스 로직을 구현했습니다. 이를 통해 각 페이지의 특성에 맞게 테이블을 파싱하여 데이터를 정확하게 추출할 수 있도록 했습니다.
  • Selenium을 사용하여 동적 크롤링을 진행할 때 페이지 로딩 속도가 저하되는 문제가 발생
  • 해결: 이 문제를 해결하기 위해 Selenium 공식 문서를 참고하여 다양한 옵션을 추가했습니다. GPU 비 활성화, 팝업 차단, 이미지 다운로드 차단 등의 옵션을 설정하여 페이지 로딩 속도를 개선시켰습니다.
MoHotel 프로젝트 상세 내용

MoHotel 프로젝트 상세 내용

📌 [개발 환경]

  • O.S/ Windows 10
  • 언어/ Java, JavaScript
  • DBMS/ MariaDB
  • 참여인원/ 4명 : 김동현
  • TOOL/
    • FrontEnd: HTML5, JS, CSS3, JQuery
    • BackEnd: Java(JDK 1.8), Servlet, Spring, Mybatis
    • IDE: IntelliJ, VisualStudio, DBeaver
    • Server: Tomcat(v8.5)
    • DataBase: MariaDB

🏠 [프로젝트 주요 기능]

  • 카카오 상담 API 를 활용하여 실시간 상담 기능을 구현했습니다. 고객이 쉽게 상담을 요청할 수 있도록 합니다.
  • ID,PW 찾기 이메일을 통해 사용자가 아이디와 비밀번호를 찾을 수 있는 기능을 제공합니다.
  • 실시간 객실 현황을 확인할 수 있는 기능을 구현했습니다. Fullcalendar API 를 활용하여 청소팀의 일정을 표시하고, 체크아웃 및 체크인 객실 번호와 청소팀의 상태를 실시간으로 확인할 수 있습니다.
  • 실시간으로 청소 요청을 보낼 수 있어 고객이 필요할 때 즉시 청소를 받을 수 있습니다.
  • 고객의 정보와 예약 정보를 확인할 수 있는 기능을 제공합니다. 유저 정보는 Excel로 출력할 수 있습니다.
  • 월별 및 일별로 객실별 예약과 매출 통계를 확인할 수 있는 기능을 제공합니다.

⛽ [프로젝트 상세 설명]

MyPage
  • 회원의 상세 정보를 확인하고 수정할 수 있는 기능을 제공합니다.
  • 프로필 아래의 비밀번호 변경 버튼을 클릭하여 모달 창으로 비밀번호를 변경할 수 있습니다.
  • 유효성 검사를 실시하여 입력한 정보의 정확성을 검증합니다.(클라이언트, 서버 두 부분 모두 유효성 검사를 실시합니다 )
  • 카카오 상담 API를 사용하여 실시간 상담 기능을 구현했습니다.
  • JavaScript와 Bootstrap을 사용하여 모달 창을 구현했습니다.
ID,PW 찾기
  • 가입 시 입력한 이메일 주소로 사용 중인 ID를 확인하고, 해당 이메일로 ID를 발송합니다.
  • 사용자가 입력한 ID를 확인하여 해당하는 이메일 주소로 임시 비밀번호를 생성하고 발송합니다.
  • 이메일 발송 기능은 구글 SMTP를 이용하여 구현되었습니다.
  • 임시 비밀번호 생성에는 UUID(Universally Unique Identifier)를 사용하여 보안성을 강화했습니다.
실시간 객실 현황
  • 실시간 객실 상태 확인을 위해 Ajax를 이용하여 동적으로 객실 상태를 업데이트하도록 구현했습니다.
  • 체크아웃하는 객실은 매일 오후 2시에 스프링 스케줄러를 활용하여 일괄적으로 퇴실 상태로 변경되며, 이후 10분 후에는 청소 요청 상태로 자동 전환됩니다.
  • 퇴실 후 청소 완료된 객실은 스케줄러를 이용하여 재실이 아닌 예약 가능 상태로 변경됩니다.
  • 쿼리문에 조건문을 추가하여 당일 체크아웃 예정인 객실은 퇴실로 자동 변경되도록 설정했습니다.
청소업체 스케줄 관리
  • 페이지 상단에는 금일 체크아웃 및 체크인 예정인 객실 번호와 청소 팀의 확인이 가능합니다.
  • 서버 측에서 금일 체크아웃 및 체크인 예정인 객실 정보를 조회하여 페이지 상단에 표시하고, 청소 팀의 정보는 데이터베이스에서 조회하여 표시합니다.
  • Fullcalendar를 이용하여 청소 팀의 일정을 시각적으로 표시합니다.
  • Fullcalendar 라이브러리를 사용하여 캘린더를 구현하고, 청소 팀의 일정은 이벤트 형태로 캘린더에 추가하여 표시합니다.
  • 페이지 우측에는 스케줄을 등록할 수 있는 기능을 제공합니다.
  • 사용자가 등록한 청소 일정은 데이터베이스에 저장되고, Fullcalendar에 이벤트로 추가됩니다. 등록된 일정은 캘린더의 이벤트를 클릭하여 삭제할 수 있습니다.
청소요청
  • 사용자는 페이지를 새로고침하지 않고도 실시간으로 청소 요청을 확인할 수 있습니다.
  • Ajax를 이용하여 서버와 비동기적으로 통신하여 새로운 청소 요청이 있을 때마다 화면을 업데이트합니다.
  • 청소가 시작되면 Ajax를 사용하여 서버에 청소 중 상태를 전달하고, 이에 따라 UI를 변경합니다.
  • 예를 들어, 청소 중인 객실은 청소 중으로 표시되고 버튼이 비활성화됩니다.
고객관리 & 엑셀 다운로드
  • 유저정보와 유저예약 정보를 확인 가능합니다.
  • 데이터베이스(DB)의 회원권한 정보를 활용하여 권한별로 다른 UI가 출력됩니다.
  • 유저정보는 Excel 파일로 출력 가능합니다.
  • Apache Poi 라이브러리를 사용하여 엑셀 다운로드 기능을 구현하였습니다.
재고관리 & 알람
  • 관리자가 '재고부족' 알림을 클릭하면 재고리스트 페이지로 이동하며, 재고부족 상품을 클릭하면 발주상태가 '발주요청'으로 변경되고 해당 업체로 '발주요청' 알림이 발송됩니다.
  • 업체가 '발주요청' 알림을 클릭하면 업체별 재고 관리 리스트로 이동 후 업체는 해당 상품을 발주할 수 있습니다.
  • 상품 판매업체는 '발주요청' 상태를 클릭하여 '발주중'으로 변경할 수 있습니다.
  • 이 기능들은 데이터베이스의 데이터를 수정하여 UI를 변경하고 웹소켓과 Ajax를 사용하여 실시간 알림을 구현합니다.
  • 스프링 스케줄러를 사용하여 매일 오전 9시에 '발주중' 상태는 '발주완료' 상태로 자동으로 변경됩니다.
  • 이 기능은 스프링 스케줄러를 사용하여 구현하였습니다.
  • '발주완료' 상태는 다시 클릭하여 '양호' 상태로 변경 가능합니다.
매출분석
  • Google Chart API를 활용하여 월별 및 일별 통계를 다양한 그래프 형태로 시각화하였습니다. 예를 들어, 선 그래프, 막대 그래프 등을 활용하여 데이터를 효과적으로 표현하였습니다.
  • 월별 및 일별로 객실별 예약 및 매출 통계를 확인할 수 있습니다.
  • 일별로 예약된 객실의 개수를 그래프로 시각화하여 표시하였습니다.
